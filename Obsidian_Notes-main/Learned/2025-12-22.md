4. NULL / NULL = **NULL**
	1. To fix this, you use the ==**COALESCE()**== function. This function basically says: "Try to do the calculation. If the result is NULL, give me 0 instead."
		```sql
		COALESCE( ROUND(SUM(p.price * u.units)::DECIMAL / SUM(u.units), 2), 0 )
		```

5. 🔥⭐ **The Rule of Thumb**
	- **Use ON:** When you want to apply a ==filter to the second table== (the Right table), but you ==still want to keep the rows== from the first table (the Left table) even if the condition fails.
	- **Use WHERE:** When you want to filter the ==**final result**== and remove rows that don't meet the criteria, regardless of the join type.
6. using `on` is faster than `using`


### **PRIMER**
 
#### **1. 8086 Architecture Overview**
- **16-bit microprocessor** (data bus: 16 bits, address bus: 20 bits).
- **Segmented memory architecture**: Allows addressing up to **1 MB** of memory (2²⁰ addresses).
- **Two main units**:
  1. **Bus Interface Unit (==BIU==)**: Fetches instructions, handles memory/I/O access.
  2. **Execution Unit (==EU==)**: Decodes and executes instructions.

---

#### **2. Registers**
The 8086 has **14 registers**, grouped into categories:
##### **1. General-Purpose Registers**:
- **AX (Accumulator)**: Primary for arithmetic operations.
- **BX (Base)**: Often used as a base pointer for memory addressing.
- **CX (Counter)**: Used in loops (e.g., `LOOP` instruction).
- **DX (Data)**: Used in I/O operations and multiply/divide.

##### **2. Pointer/Index Registers**:
- **SP (Stack Pointer)**: Points to the top of the stack.
- **BP (Base Pointer)**: Points to data in the stack segment.
- **SI (Source Index)**: Used in string operations as a source pointer.
- **DI (Destination Index)**: Used in string operations as a destination pointer.

##### **3. Segment Registers** (16-bit):
- **CS (Code Segment)**: Holds the base address of the code segment.
- **DS (Data Segment)**: Base address for data variables.
- **SS (Stack Segment)**: Base address for the stack.
- **ES (Extra Segment)**: Additional data segment (e.g., for string operations).

##### **4. Control Registers**:
- **IP (Instruction Pointer)**: Points to the next instruction to execute (offset in CS).
- **Flags Register**: Contains status flags (e.g., **CF, ZF, SF, OF**).

---

#### **3. Memory Segmentation & Addressing**
- **Physical Address Calculation**:
  ```python
  Physical Address = (Segment Register × 16) + Offset
  ```
  EG: CS = 1234H, IP = 5678H  
	→ Physical address = 1234H × 10H + 5678H = **179B8H**

- ==**Why Segmentation?**:==
  - Allows access to ==1 MB memory using 16-bit registers.==
  - Segments can overlap (e.g., `CS`, `DS`, `SS` can point to the same physical address).
  - The segment register provides a 16-bit base address (shifted left by 4 bits), and the offset is added to reach the final 20-bit physical address.
  - ==**20 Bits:** This allows addressing from 0x00000 to 0xFFFFF (1 MB of memory).==

---

#### **4. Addressing Modes**
The 8086 supports **7 addressing modes** (critical for interviews):
1. **Immediate**: Operand is part of the instruction (e.g., `MOV AX, 05H`).
2. **Register**: Operand is in a register (e.g., `MOV AX, BX`).
3. **Direct**: Operand’s offset is in the instruction (e.g., `MOV AX, [1234H]`).
4. **Register Indirect**: Address is in a register (e.g., `MOV AX, [BX]`).
5. **Based**: Uses `BX` or `BP` as a base register (e.g., `MOV AX, [BX+04H]`).
6. **Indexed**: Uses `SI` or `DI` as an index (e.g., `MOV AX, [SI+02H]`).
7. **Based Indexed**: Combines base and index registers (e.g., `MOV AX, [BX+SI+08H]`).

---

#### **5. Interrupts**
- **256 Interrupts** (0 to 255) stored in the **Interrupt Vector Table (IVT)** at `0000:0000` to `0000:03FF`.
- **Types**:
  - **Hardware Interrupts**: Triggered by external devices (e.g., `INTR`, `NMI`).
  - **Software Interrupts**: Generated by instructions (e.g., `INT 21H` for DOS services).

---

#### **6. Flag Register**
- **Status Flags**:
  - **CF (Carry Flag)**: Set if arithmetic operation generates a carry/borrow.
  - **ZF (Zero Flag)**: Set if result is zero.
  - **SF (Sign Flag)**: Set if result is negative.
  - **OF (Overflow Flag)**: Set if signed arithmetic overflow occurs.
- **Control Flags**:
  - **TF (Trap Flag)**: Used for single-step debugging.
  - **IF (Interrupt Flag)**: Enables/disables hardware interrupts.
  - **DF (Direction Flag)**: Controls string direction (`CLD` clears, `STD` sets).

---

#### **7. Important Instructions**
- **Data Transfer**: `MOV`, `PUSH`, `POP`, `XCHG`.
- **Arithmetic**: `ADD`, `SUB`, `MUL`, `DIV`, `CMP`.
- **Control Transfer**: `JMP`, `CALL`, `RET`, `LOOP`.
- **String Operations**: `MOVSB`, `CMPSB`, `STOSB`, `LODSB` (use `REP` prefix).
- **Interrupts**: `INT`, `IRET`.

---

#### **8. I/O Addressing**
- **Isolated I/O**: Uses `IN` and `OUT` instructions (port addresses: 8-bit or 16-bit).
- **Memory-Mapped I/O**: Treats I/O ports as memory locations (rare in 8086).

---

#### **9. Clock Speed & Pipelining**
- **Clock Speed**: 5 MHz (original), later 8 MHz and 10 MHz.
- **Pipelining**: BIU prefetches instructions into a **6-byte queue** while the EU executes them.

**Q:** *Write assembly code to copy a block of 10 bytes from source (DS:SI) to destination (ES:DI).*  
**A:**

```asm
REP MOVSB        ; Repeat MOVSB until CX=0
```

### **Addressing Modes**

|**Addressing Mode**|**Description**|**Example**|
|---|---|---|
|**1. Immediate**|The operand is directly specified in the instruction.|`MOV AX, 5` (Move the value 5 into AX).|
|**2. Direct**|The effective address of the operand is explicitly mentioned in the instruction.|`MOV AX, [1234H]` (Move the data at address 1234H to AX).|
|**3. Register**|The operand is in a CPU register, and the instruction specifies the register directly.|`MOV AX, BX` (Move the value in BX to AX).|
|**4. Register Indirect**|The address of the operand is in a register, and the instruction refers to that register.|`MOV AX, [BX]` (Move the value at the address in BX to AX).|
|**5. Indexed**|The effective address is the sum of a constant (index) and a register value.|`MOV AX, [BX + 10H]` (Add 10H to BX’s content to get the effective address).|
|**6. Base**|Similar to Indexed, but uses a base register for addressing instead of an index register.|`MOV AX, [BP + 20H]` (Base pointer BP with offset 20H).|
|**7. Based Indexed**|The effective address is calculated using both base and index registers.|`MOV AX, [BX + SI]` (Sum of BX and SI gives the effective address).|
|**8. Relative**|The effective address is relative to the Program Counter (PC).|`JMP 1234H` (Jump to address relative to the current PC value).|
|**9. Stack**|The operand is implied to be on the top of the stack.|`POP AX` (Remove the top stack element and store it in AX).|

---

#### **Detailed Examples**

6. **Immediate Addressing Mode**
    
    - **Definition**: The operand is part of the instruction itself.
    - **Example**:
        
        ```
        MOV AX, 25
        ```
        
        AX gets the constant value 25.
7. **Direct Addressing Mode**
    
    - **Definition**: The effective address is directly mentioned in the instruction.
    - **Example**:
        
        ```
        MOV AX, [5000H]
        ```
        
        The value stored at memory location 5000H is moved to AX.
8. **Register Addressing Mode**
    
    - **Definition**: The operand is located in a specific CPU register.
    - **Example**:
        
        ```
        MOV AX, BX
        ```
        
        The content of BX is copied into AX.
9. **Register Indirect Addressing Mode**
    
    - **Definition**: The address of the operand is stored in a register.
    - **Example**:
        
        ```
        MOV AX, [BX]
        ```
        
        The data at the address stored in BX is copied into AX.
10. **Indexed Addressing Mode**
    
    - **Definition**: The effective address is obtained by adding an index (constant) to a register.
    - **Example**:
        
        ```
        MOV AX, [BX + 10H]
        ```
        
        Add 10H to BX, and move the data at the resulting address into AX.
11. **Base Addressing Mode**
    
    - **Definition**: Similar to Indexed, but uses a base register like BP (Base Pointer).
    - **Example**:
        
        ```
        MOV AX, [BP + 20H]
        ```
        
        Add 20H to BP, and move the data from that address into AX.
12. **Based Indexed Addressing Mode**
    
    - **Definition**: The effective address is obtained by adding a base register and an index register.
    - **Example**:
        
        ```
        MOV AX, [BX + SI]
        ```
        
        Add the values of BX and SI, and move the data at that effective address into AX.
13. **Relative Addressing Mode**
    
    - **Definition**: The effective address is calculated relative to the current instruction address (PC).
    - **Example**:
        
        ```
        JMP 100H
        ```
        
        Jump to an address that is offset by 100H from the current PC.
14. **Stack Addressing Mode**
    
    - **Definition**: Operates on the stack using implicit addressing (e.g., Top of Stack).
    - **Example**:
        
        ```
        POP AX
        ```
        
        Remove the value from the top of the stack and place it into AX.

---

#### **Key Differences Between Addressing Modes**

|**Mode**|**Address Calculation**|**Key Use Case**|
|---|---|---|
|Immediate|No address calculation; the value is directly in the instruction.|Used for constants or literals.|
|Direct|Address specified in the instruction.|Access specific memory locations.|
|Register|Operand is directly in a CPU register.|Fast operations within registers.|
|Register Indirect|Address is in a register, and the operand is in memory at that address.|Flexible for memory operations.|
|Indexed|Address = Register + Offset.|Used in array or table processing.|
|Base|Address = Base Register + Offset.|Access stack frames or structured data.|
|Based Indexed|Address = Base Register + Index Register.|Complex data structures, multi-d arrays.|
|Relative|Address = Program Counter + Offset.|Jumps and branches in code.|
|Stack|Operates implicitly on the stack.|Push and Pop operations.|
